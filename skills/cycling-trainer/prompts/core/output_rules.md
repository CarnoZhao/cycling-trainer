# 输出通用规则

> 被所有 command prompts 引用

## "今天"的定义

- **today** 字段表示用户发送命令的当天日期
- **第1天 = today**，不是明天！
- 如果今天是周五，那么【近3天】应该是：**周五、周六、周日**
- ❌ 错误：把今天当成"已结束"，从明天开始规划
- ✅ 正确：即使今天应该休息，也要在计划中列出"今天 — 🛌 完全休息"

## 日期核对要求

**输出前必须核对日期和星期几的对应关系**：

```
今天是 {today} ({today_weekday})
明天是 {tomorrow} ({tomorrow_weekday})
后天是 {day_after_tomorrow} ({day_after_tomorrow_weekday})
```

**格式要求**: `3/12 (周四)` — 日期和星期几必须匹配

## 本周进度计算

- 从周一开始到 **yesterday**（昨天）为止，统计已完成的强度课数量
- **today 的活动数据是否存在？**
  - 如果存在 → 今天已经骑过了，计入已完成
  - 如果不存在 → 今天还没骑，不计入已完成
- 本周剩余可安排天数 = 从 today 到本周日（含）

## 安排逻辑

1. **明确声明今天是哪一天**："今天是 {日期} ({星期})"
2. **检查今天是否已经骑行**：查看 today 是否有活动数据
3. **判断当前周期位置**：根据 CYCLE_MEMORY 或历史数据分析
4. **从今天起向后规划7天**，每一天都要列出（包括休息日）
5. **本周剩余强度课** = 2节 - 已完成的数量（不含今天，除非今天已骑）
6. **按周期周位置确定训练难度**：第1周入门、第2周进阶、第3周峰值、第4周恢复
