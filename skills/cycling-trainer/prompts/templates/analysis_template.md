# 分析报告通用模板

> 被 commands/status_analysis.md、form_analysis.md、trend_analysis.md、week_summary.md、latest_ride_analysis.md 引用

## 标准报告头部

```
══════════════════════════════════════
        🚴 {报告标题}
══════════════════════════════════════
📅 {{日期}} | {{星期}}

👤 Carno  ⚡ FTP {ftp}w  📊 TSB {tsb}
```

## 状态报告格式

```
┌─────────────────────────────────────┐
│  CTL (42天)  │  {{值}}  │  {{趋势}}   │
│  ATL (7天)   │  {{值}}  │  {{疲劳度}} │
│  TSB         │  {{值}}  │  {{状态}}   │
└─────────────────────────────────────┘

📊 本周: {{次数}}次 / {{时长}}h / 负荷{{值}}

📝 分析:
{{状态分析文字}}

💡 建议:
{{具体建议}}
══════════════════════════════════════
```

## 走势报告格式

```
【状态走势分析】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 对比基准
   心率区间: [具体范围]
   持续时间: [范围]分钟
   样本数量: [N]个间歇

📈 巅峰 vs 当前
   
   巅峰表现:
   • 日期: [YYYY-MM-DD]
   • 功率: [XXX]w
   • 活动: [名称]
   
   当前表现:
   • 日期: [YYYY-MM-DD]  
   • 功率: [YYY]w
   • 活动: [名称]
   
   变化: -[ZZ]w (-[P]%)

🎯 状态评估: [稳定/轻微下滑/明显下滑/严重下滑]

🔍 原因分析:
   [基于数据趋势和训练历史的专业分析]

💡 恢复建议:
   1. [具体建议1]
   2. [具体建议2]
   3. [具体建议3]

⏱️ 预计恢复时间: [X周]
```

## 单次骑行报告格式

```
📊 **骑行概览**
• 类型: {Z2/Tempo/Threshold/VO2max等}
• 时长: {xx}min · 距离: {xx}km
• TSS: {xxx} · IF: {x.xx}

📈 **执行质量**
• 功率稳定性: {评价}
• 心率响应: {评价}
• 脱耦率: {xx}% ({评价})

🔥 **间歇分析** (如适用)
• 目标达成率: {xx}%
• 组间一致性: {评价}
• 疲劳趋势: {评价}

💡 **教练点评**
{具体建议}
```
